# X-Mobility Navigator プログラム解説

## 概要
このプログラムは、NVIDIA社のロボットナビゲーションシステム「X-Mobility Navigator」のROS2ノード実装です。TensorRTを使った深層学習ベースの自律走行システムで、カメラ画像とルート情報から車両の制御コマンドを生成します。

## 主な機能

### 1. **センサーデータの購読**
- **カメラ画像** (`/front_stereo_camera/left/image_raw`): 前方ステレオカメラの左画像
- **オドメトリ** (`/chassis/odom`): ロボットの現在速度・位置情報
- **ルート情報** (`/route`): 地図ベースの計画経路
- **ゴール地点** (`/goal_pose`): 目標位置

### 2. **TensorRT推論**
- 深層学習モデル（.engineファイル）をTensorRTで高速実行
- **入力**:
  - カメラ画像（正規化済み、C×H×W形式）
  - ルートベクトル（20点のウェイポイント）
  - 自車速度
  - 前回の行動（時系列連続性）
  - 履歴状態（RNN/LSTMの隠れ状態）
  - サンプリング変数（確率的行動生成）

- **出力**:
  - 行動コマンド（速度・角速度）
  - 予測パス
  - 更新された履歴状態

### 3. **制御コマンドの発行**
- **速度コマンド** (`/cmd_vel`): ロボットへの移動指令
  - `linear.x`: 前進速度 (m/s)
  - `angular.z`: 角速度 (rad/s)
- **可視化パス** (`/x_mobility_path`): RViz等で表示可能な経路

### 4. **動作モード**
- **マップベースモード**: 外部から提供されたルートに従って走行
- **マップレスモード**: ゴール地点への直線ルートを自動生成

## 処理フロー

```
起動
 ↓
1. TensorRTモデルのロード (.engine ファイル)
 ↓
2. センサーデータの購読開始
 ↓
3. タイマー起動 (0.2秒ごと = 5Hz)
 ↓
4. 各タイマー実行時:
   - マップレスモードならルート生成
   - 入力データの準備確認
   - GPU上でTensorRT推論実行
   - 制御コマンドの発行
   - パスの可視化
```

## 重要なクラスとメソッド

### `XMobilityNavigator` クラス
メインのROS2ノードクラス。

#### `__init__()`
- パラメータ設定（モデルパス、マップレスモード）
- サブスクライバー/パブリッシャーの初期化
- タイマー設定（5Hz）
- 内部状態変数の初期化

#### `load_model()`
- TensorRTエンジンファイル（.engine）を読み込み
- 推論用の実行コンテキストを作成

#### `inference()`
- 定期的に呼ばれる推論メインループ（0.2秒ごと）
- データ準備 → TensorRT推論 → 結果発行

#### `_trt_inference()`
- TensorRTでの実際の推論処理
- GPU上でメモリ確保 → データ転送 → 推論実行 → 結果取得

#### コールバック関数
- `image_callback()`: カメラ画像の前処理
- `odom_callback()`: 速度情報の取得
- `goal_callback()`: ゴール設定時に履歴リセット
- `route_callback()`: マップベースルートの座標変換

#### `compose_mapless_route()`
- マップレスモード用の直線ルート生成
- 現在位置からゴールまでを1mセグメントで補間

#### `publish_action()`
- 推論結果の制御コマンドをロボットへ送信

#### `publish_path()`
- ルートをRViz可視化用にPath形式で発行

### ユーティリティ関数

#### `upsample_points(start, goal, max_segment_length)`
- 2点間を指定距離以下のセグメントに分割
- 直線補間で中間ウェイポイントを生成

## 技術的ポイント

### TensorRTの活用
- NVIDIAの高速推論エンジンで深層学習モデルを実行
- GPU上でリアルタイム処理（5Hz）
- モデルは事前に最適化された.engineファイル形式

### 座標変換（TF2）
- ロボット座標系（base_link）への変換
- 異なる座標系間でのルート/ゴールの位置合わせ

### 時系列処理
- 履歴状態（history）とサンプリング変数（sample）の保持
- RNN/LSTMによる過去情報の活用で滑らかな制御

### GPU処理
- PyCUDAでホスト↔デバイス間のデータ転送
- 非同期処理のためのCUDAストリーム

## 実行方法
```bash
# ROS2ノードとして起動
ros2 run <package_name> <node_name>

# パラメータ指定例
ros2 run <package_name> <node_name> \
  --ros-args \
  -p runtime_path:=/path/to/model.engine \
  -p is_mapless:=true
```

## 応用例
- 自律走行ロボット
- 配送ロボット
- 倉庫内搬送AGV
- 屋外ナビゲーション

このシステムは、カメラ画像を使ったend-to-end学習により、人間がプログラムした明示的なルールではなく、ニューラルネットワークが環境を理解して運転判断を行う点が特徴です。
